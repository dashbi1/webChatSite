## Slide 2 — 选题与分组

**选题**：工大圈子（hit-circle）

一个面向哈工大学生的**轻量级校园社交平台**，支持发动态、加好友、互动、实时私聊，并内置完整的反滥用风控体系。

**核心特点**：
- 三端同构：Web H5 + Android APK + 微信小程序（uni-app 一套代码）
- 开放表达 + 好友深度互动：所有帖子圈子内可见，但点赞 / 评论 / 私聊仅限好友
- 未雨绸缪的反滥用架构：从上线第一天就面对黑产级威胁建模

## Slide 3 — 对题目的理解

**项目背景**

在当前社交媒体碎片化的环境下，校园学生缺乏一个**专属的、轻量级的交流空间**。工大圈子旨在构建一个以"圈子"为核心的社区，让学生自由表达观点、结识同校同学、进行即时交流。

**现实意义**

- 填补校园社交空白：相比微信朋友圈的熟人壁垒，提供开放的同校社区
- 降低表达门槛：手机号 / 邮箱注册即可使用，无需学号认证
- 可控的内容生态：完整的举报 + 管理员机制 + 反滥用引擎，保证健康氛围

**目标用户**

| 角色 | 描述 |
|------|------|
| 普通用户 | 哈工大学生（实际不限制），浏览动态、发帖、社交、私聊 |
| 管理员 | 平台运营人员，负责用户状态、违规内容、账号簇、申诉处理 |

**需求出发点**

1. **开放表达**：所有用户发布的动态对圈子内全部成员可见
2. **好友深度互动**：通过好友机制筛选社交深度（仅好友可点赞、评论、私聊）
3. **轻量即时通讯**：好友间支持实时私聊，可转发圈子内容
4. **平台自我治理**：举报 + 管理员后台 + 风控引擎三位一体

## Slide 5 — 非功能需求（NFR）

| NFR 维度 | 目标 | 为何要这样设计 |
|---------|------|---------------|
| **性能** | API 响应 < 500ms；信息流首屏 < 2s（缓存命中瞬开） | 100 人规模日活；Supabase 美国区存在 150ms 延迟，需要前端缓存 + 骨架屏 |
| **可用性** | 99% 可用；7×24 小时 | 单 VPS + PM2 守护 + Nginx 反代；非高可用但保证基本稳定 |
| **安全性** | HTTPS 全站 + JWT 认证 + bcrypt 密码哈希 + Supabase RLS | 学生真实社交数据，防截获 / 防越权 |
| **反滥用** | 上线首日即 enforce 模式；四档风险分自动化处置 | 小规模但设计面向黑产级威胁（秒拨 IP / 改机 / 接码） |
| **跨平台** | Web + Android + 微信小程序 | uni-app 一套代码编译多端，最大化用户触达 |
| **可维护性** | 前后端分离 + RESTful + 单一职责模块 | 3 人团队，需清晰分工与并行开发 |
| **易用性** | 注册 → 浏览 → 发帖 3 步内完成 | 校园用户对产品容错度低，首次体验必须流畅 |
| **可观测性** | PM2 + 结构化日志 + 风险事件审计 | 便于定位线上问题、排查风控误判 |

**为何特别强调反滥用 NFR**：系统面向"未雨绸缪"威胁模型设计，一旦被职业黑产盯上，**只调参数不改架构**即可升级防御——这是架构决策的核心约束。

## Slide 6 — 系统的体系结构构思

> 采用经典**三层 + 云服务托管**架构，兼顾开发速度与后期扩展。

```
┌─────────────────────────────────────────────────────┐
│                  客户端（uni-app）                   │
│            Web H5              Android APK          │
└───────────────────────┬─────────────────────────────┘
                        │ HTTPS / WSS
                        ↓
┌─────────────────────────────────────────────────────┐
│               Cloudflare（CDN + WAF）                │
│          Turnstile 人机验证 + Rate Limiting           │
└───────────────────────┬─────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│                   VPS（2C4G，日本）                   │
│  ┌─────────────────────────────────────────────┐    │
│  │        Nginx（反代 + SSL 终止 + 真实 IP）      │    │
│  └─────────────────┬───────────────────────────┘    │
│                    ↓                                 │
│  ┌─────────────────────────────────────────────┐    │
│  │           Node.js（Express + Socket.io）      │    │
│  │  路由层 → 中间件 → 业务服务 → 反滥用引擎        │    │
│  │   （auth / posts / friends / chat / admin）  │    │
│  │   （riskEngine / fingerprint / cluster）     │    │
│  └────┬──────────────────┬───────────────┬──────┘   │
└───────┼──────────────────┼───────────────┼──────────┘
        ↓                  ↓               ↓
  ┌──────────┐      ┌──────────┐    ┌──────────────┐
  │ Supabase │      │ Upstash  │    │ Resend Mail  │
  │  PG + 认证│      │  Redis   │    │  邮箱验证码   │
  │ + Storage│      │ （限流）  │    │              │
  └──────────┘      └──────────┘    └──────────────┘
```

**关键架构决策**：
- **Supabase 托管数据库**：省去运维，免费套餐满足 100 人规模
- **Upstash Redis**：无需本机装 Redis，云端免费 10k 命令/天
- **Cloudflare Free**：免费 Turnstile、WAF、DDoS 防护
- **PM2 守护 + Nginx 反代**：进程不死 + 静态资源高效分发

- VPS：2C4G 日本节点，Ubuntu 22.04

- 进程管理：PM2（ecosystem.config.js）
- SSL：Let's Encrypt 免费证书
- CDN / WAF：Cloudflare Free
- 数据库部署：Supabase free版本
- redis部署：Upstash redis 免费版本
- 邮箱验证码发送：Resend + 自有域名
  

**主要技术栈**

| 层 | 技术选型 | 作用 |

| 认证 | Supabase Auth + JWT + bcrypt | 邮箱密码登录 |
| 文件存储 | Supabase Storage | 头像 / 图片 / 视频 |
| 限流缓存 | Upstash Redis | IP / 邮箱 / 设备限流 |
| 人机验证 | Cloudflare Turnstile | 防脚本注册 |
| 邮件服务 | Resend | 验证码投递 |
| 指纹采集 | FingerprintJS + Capacitor 插件 | 浏览器 + APK 双端指纹 |
| 测试框架 | Jest + supertest + Playwright | 单元 + 集成 + E2E |
