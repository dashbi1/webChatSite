# 工大圈子 - 迭代3 实施计划

## 迭代3 目标
基于迭代2 的联调反馈，补全基础功能、修复数据一致性问题、全面升级 UI 体验。

|------|------|--------|------|

| US23 | 给被封禁用户发消息时前端弹出提示 | 2 | 迭代2 联调反馈 |

### 用户故事详情

#### US22 - 全局自动刷新
- **角色**：用户
- **描述**：作为一个用户，我想要页面数据自动刷新，以便不用手动操作就能看到最新内容
- **验收条件**：
  - 信息流每 30 秒自动轮询
  - 所有列表页支持下拉刷新
  - 轮询时不打断用户操作（静默刷新）

#### US23 - 封禁用户发消息提示
- **角色**：用户
- **描述**：当我给被封禁的好友发消息时，应该收到明确的提示
- **验收条件**：
  - 聊天页发送失败时弹出 Toast "对方账号已被封禁"
  - Socket `chat:error` 事件在前端正确展示

#### US24 - 修改密码
- **角色**：用户
- **描述**：作为一个用户，我想要修改密码以保护账号安全
- **验收条件**：
  - "我的" → "修改密码"入口
  - 输入旧密码 + 新密码 + 确认新密码
  - 旧密码验证通过才能修改
  - 修改成功后需重新登录

#### US25 - 聊天消息时间标签
- **角色**：用户
- **描述**：聊天页中的消息应显示发送时间
- **验收条件**：
  - 超过 5 分钟间隔的消息之间显示时间分隔线
  - 格式：今天的显示"HH:mm"，昨天的显示"昨天 HH:mm"，更早的显示"MM/DD HH:mm"

#### US26 - 页面间数据一致性
- **角色**：用户
- **描述**：在帖子详情页点赞/评论后，返回首页数据应同步更新
- **验收条件**：
  - 从详情页返回后，首页对应帖子的点赞数/评论数已更新
  - 从编辑资料页返回后，"我的"页面头像/昵称已更新

#### US27 - 评论准实时更新
- **角色**：用户
- **描述**：帖子详情页的评论应自动更新
- **验收条件**：
  - 发表评论后立即显示在列表中（不需要退出重进）
  - 每 15 秒自动轮询新评论

#### US28 - 前端 UI 全面升级
- **角色**：用户
- **描述**：提升整体视觉体验，清爽简约风格
- **验收条件**：
  - 主色调 #4A90D9 蓝色保留
  - 登录/注册页重新设计
  - 首页信息流卡片样式优化
  - 个人中心页重新设计
  - 聊天页样式优化
  - 统一间距、圆角、阴影、字体层级
  - 管理后台不改

## Phase 划分

| Phase | 内容 | 用户故事 | 预估工作量 |
|-------|------|---------|-----------|
| Phase 11 | 基础补全 | US24 + US25 + US23 | 5h |
| Phase 12 | 数据一致性 + 刷新 | US22 + US26 + US27 | 6h |
| Phase 13 | UI 全面升级 | US28 | 8h |

**开发顺序**：先 Phase 13（UI），再 Phase 11，最后 Phase 12。
（用户要求先改 UI）

## 迭代3 开发策略

1. UI 改版先做，后续功能改动在新 UI 上进行
2. 每个 Phase 完成后运行全量测试，确保不破坏已有功能
3. Phase 11/12 写新测试，Phase 13 运行已有测试回归验证
4. 管理后台不在本迭代 UI 改版范围内
