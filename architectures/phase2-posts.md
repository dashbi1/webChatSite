# Phase 2: 信息流 + 发帖

## 实现状态：已完成

## 对应用户故事
- US01 - 浏览信息流
- US02 - 发布文字帖子
- US18 - 删除/编辑帖子

## API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/posts?page=&limit=` | 获取信息流（分页，时间倒序） |
| POST | `/api/posts` | 发布帖子（content，1-1000 字） |
| PUT | `/api/posts/:id` | 编辑帖子（仅作者可操作） |
| DELETE | `/api/posts/:id` | 删除帖子（作者或管理员） |

## 技术实现
- **分页**：offset + limit，默认每页 20 条
- **信息流数据**：帖子 + 作者信息 + 好友状态 + 点赞状态（批量查询，非逐条）
- **联表查询**：因 Supabase JS SDK 的 embedded select 兼容问题，改为分步查询（先查帖子，再批量查作者/好友/点赞）
- **编辑标记**：编辑后 `is_edited` 设为 true

## 关键文件
- `server/src/routes/posts.js` — 帖子路由（含点赞/评论，Phase 4 共用）

## 测试覆盖（11 用例）
- 发帖：正常发帖、空内容、纯空格、超长内容
- 信息流：时间倒序、他人可见、包含作者信息
- 编辑：编辑自己的、编辑他人的（403）
- 删除：删除自己的、删除他人的（403）
