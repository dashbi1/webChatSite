# 工大圈子 - 迭代2 实施计划

## 迭代2 目标
基于迭代1 的用户反馈，增强内容形式（图片/视频）、完善管理功能、优化用户体验。

## Phase 总览

| Phase | 内容 | 用户故事 | 预估工作量 | 负责人 |
|-------|------|---------|-----------|--------|
| Phase 7 | 资料编辑 + 图片帖 + 帖子管理 | US16 + US15 + US18 | 11h | 陈明 + 刘洋 |
| Phase 8 | 管理员后台 + 举报 | US19 + US20 + US17 | 10h | 陈明 + 张晨 |
| Phase 9 | 热帖排行 + 转发到私聊 | US14 + US13 | 10h | 刘洋 + 张晨 |
| Phase 10 | 视频帖 + 性能优化 | US21 + 优化 | 8h | 全组 |

## Phase 详细说明

### Phase 7：资料编辑 + 图片帖 + 帖子管理

**核心价值**：解决迭代1 反馈中最强烈的两个问题（F02 资料不可编辑、F03 纯文字太单调），同时补全帖子编辑/删除的前端入口。

| 故事 | 功能 | 后端 | 前端 | 测试 |
|------|------|------|------|------|
| US16 | 修改头像、昵称、学院、年级 | PUT /api/users/me（已有）+ 头像上传 | 资料编辑页 | 上传+更新 |
| US15 | 发帖时附带图片（最多9张） | Supabase Storage + media_urls 字段 | 发帖页图片选择+预览 | 图片上传+展示 |
| US18 | 自己的帖子可编辑/删除 | API 已有 | PostCard 加操作菜单 | 前端联调验证 |

**技术要点**：Supabase Storage 首次引入，需先 spike 验证上传流程。

### Phase 8：管理员后台 + 举报

**核心价值**：提供内容管理能力，确保平台健康运营。

| 故事 | 功能 | 后端 | 前端 | 测试 |
|------|------|------|------|------|
| US19 | 管理员封禁/解封用户 | API（已有基础） | 独立管理后台 | 封禁+解封 |
| US20 | 管理员删除帖子 | API（已有基础） | 管理后台帖子列表 | 管理员删帖 |
| US17 | 用户举报帖子/用户 | API | 举报按钮+弹窗 | 举报+处理 |

**技术要点**：管理后台为独立网页应用，使用随机 URL 路径（非 /admin），防止被猜测攻击。

### Phase 9：热帖排行 + 转发到私聊

**核心价值**：提升内容发现效率，增加社交互动方式。

| 故事 | 功能 | 后端 | 前端 | 测试 |
|------|------|------|------|------|
| US14 | 按热度（点赞+评论数）排序的帖子列表 | 热度计算 API | 热帖 Tab | 排序逻辑 |
| US13 | 帖子详情页分享到私聊 | 消息 type=post_share | 好友选择+发送 | 转发+展示 |

### Phase 10：视频帖 + 性能优化

**核心价值**：丰富内容形式，提升整体体验。

| 故事 | 功能 | 后端 | 前端 | 测试 |
|------|------|------|------|------|
| US21 | 发帖时附带视频 | Supabase Storage 视频上传 | 视频选择+播放器 | 上传+播放 |
| 优化 | 加载速度、骨架屏、图片懒加载 | 接口合并/缓存 | 体验优化 | 性能验证 |

## 迭代2 开发策略（基于迭代1 回顾改进）

1. **每个 Phase 完成后立即前端联调**，不再后端全做完才测前端
2. **每个 Phase 必须有测试**，保持 80%+ 覆盖率
3. **图片/视频上传先做 spike**，验证 Supabase Storage 方案后再正式开发
4. **管理后台最小 MVP**，先做核心的封禁+删帖，不搞复杂的数据看板

## 验收标准

- [ ] 9 个用户故事全部实现
- [ ] 测试用例 80+ 个，全部通过
- [ ] 前端所有页面浏览器联调通过
- [ ] 管理后台可独立访问并操作
- [ ] 图片帖可正常上传、展示
