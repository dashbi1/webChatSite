# 工大圈子 - 项目概述

## 项目背景

工大圈子是一个面向哈尔滨工业大学学生的校园社交平台。在当前社交媒体碎片化的环境下，
校园学生缺乏一个专属的、轻量级的交流空间。工大圈子旨在构建一个以"圈子"为核心的
社区，让学生能够自由表达观点、结识同校同学、进行即时交流。

## 核心价值

- **开放表达**：所有用户发布的动态对圈子内全部成员可见，信息透明流通
- **好友深度互动**：通过好友机制筛选社交深度——只有好友才能点赞、评论、私聊
- **轻量即时通讯**：好友间支持实时私聊，可转发圈子内容

## 目标用户

| 角色 | 描述 |
|------|------|
| **普通用户** | 哈工大学生（实际上任何手机号注册的用户），浏览动态、发帖、社交 |
| **管理员** | 平台管理人员，负责维护社区秩序，管理用户状态 |

## 系统边界

### 包含（Scope In）

- 手机号注册/登录（短信验证码）
- 发布动态（文字 → 图文 → 视频，渐进实现）
- 信息流浏览（按时间倒序）
- 好友系统（搜索、申请、同意/拒绝）
- 好友互动（点赞、评论）
- 转发帖子（通过私聊发送给好友）
- 实时私聊（WebSocket）
- 热帖排行
- 举报功能
- 站内通知（好友请求、评论、点赞提醒）
- 管理员：封禁/解封用户、删除帖子
- 个人资料管理（昵称、头像、学院/专业、年级）

### 不包含（Scope Out）

- 学生身份验证（不校验学号/学校邮箱）
- 手机推送通知（第一阶段不做）
- 群聊功能
- 付费/打赏功能

## 非功能需求（NFR）

| NFR | 目标 | 说明 |
|-----|------|------|
| **性能** | 接口响应 < 500ms | 100 人规模，Supabase 免费套餐足够 |
| **可用性** | 99%+ | VPS 单机部署，非高可用，但保证基本稳定 |
| **安全性** | 数据加密传输 | HTTPS + JWT token 认证 + 密码哈希 |
| **跨平台** | Web + Android + 微信小程序 | uni-app 一套代码编译多端 |
| **可维护性** | 代码结构清晰 | 前后端分离，RESTful API |
| **易用性** | 3 步内完成核心操作 | 注册→浏览→发帖 流程简洁 |
